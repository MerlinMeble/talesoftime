Tales of Time: RPG Management SystemWelcome to the Tales of Time project. This is a Flask-based web application designed to manage characters, items, and quests for a tabletop RPG. This repository serves as your primary workspace for implementing the SQL Data Access Layer and understanding relational database architecture.🛠 Technology StackLanguage: Python 3.xWeb Framework: FlaskDatabase: SQLite 3Data Access: Raw SQL (Repository Pattern)🚀 Getting StartedFollow these steps in sequence to set up your local development environment.1. Clone the RepositoryOpen your terminal and run:git clone [https://github.com/](https://github.com/)[YOUR-USERNAME]/talesoftime.git
cd talesoftime
2. Initialise the DatabaseCreate the database skeleton and tables defined in your models.python -c "from models.models import init_db; init_db(); print('Database Skeleton Created!')"
3. Seed the Game DataPopulate your lookup tables and core entities with starting data.python database/seed.py
4. Launch the ApplicationStart the Flask development server.python app.py
Access the app at: http://127.0.0.1:5000📝 Core Development TasksDuring this module, you are responsible for implementing the following components:Phase 1: Database Schema (database/schema.sql)Define the relational structure, including:Lookup Tables: Classes, Species, Alignments, Regions.Core Entities: Characters, Items, Quests.Join Tables: Inventory (Many-to-Many), CharacterQuests.Phase 2: Connection Logic (models/models.py)Configure the SQLite bridge:Implementing get_db() with sqlite3.Row for dictionary-style access.Enforcing PRAGMA foreign_keys = ON for data integrity.Defining the _SCHEMA string for self-healing initialization.Phase 3: Data Repositories (repositories/repositories.py)Isolate SQL logic from the UI:Using Parameterized Queries (?) to prevent SQL Injection.Writing explicit JOIN statements to retrieve readable data.Implementing business logic for item stacking in the InventoryRepository.🛠 Useful Terminal CommandsCommandPurposegit statusCheck which files have been modified.git add .Stage all changes for a commit.git commit -m "Message"Save a snapshot of your progress.git push origin mainSync your work with GitHub.CTRL + CStop the Flask web server.⚠️ TroubleshootingDatabase Locked: Ensure you have closed any external database browser tools (like DB Browser for SQLite) before running the app.Foreign Key Errors: This happens if you try to delete a row that is referenced by another table. Check your ON DELETE CASCADE settings.Missing Data: If your UI is empty, ensure you ran python database/seed.py successfully.Developed as part of the Web Development & Database Design curriculum.
